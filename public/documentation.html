This page will give instructions of the features and applications of my movie-api project

Get Movies

Description: Retrieves a list of movies.

Endpoint URL: /movies

Method Type: GET

Query Parameters: None

Request Body: None (This endpoint does not expect any data in the request body)

Example Request:
    GET /movies

Response Data:

Format: JSON
    Example:
    [
      {
        "id": 1,
        "title": "Inception",
        "genre": "Sci-Fi",
        "releaseYear": 2010
      },
      {
        "id": 2,
        "title": "The Shawshank Redemption",
        "genre": "Drama",
        "releaseYear": 1994
      },
      // ... additional movie objects
    ]

Status Codes:

  • 200 OK: Successful request, returns a list of movies.
  • 404 Not Found: The requested resource (movies) was not found.

Notes:

  • The response contains an array of movie objects, each providing information such as movie ID, title, genre, and release year.
  • Ensure that the request does not include any query parameters, as they are not applicable to this endpoint.

Get Movie by Title

Description: Retrieves a movie by its title.

Endpoint URL: /movies/:title

Method Type: GET

Query Parameters: None

Request Body: None (This endpoint does not expect any data in the request body)

Example Request:
  GET /movies/Inception

Response Data:

Format: JSON
  Example:
  {
    "id": 1,
    "title": "Inception",
    "genre": "Sci-Fi",
    "releaseYear": 2010
  }

Status Codes:

  • 200 OK: Successful request, returns the movie with the specified title.
  • 404 Not Found: The requested movie with the specified title was not found.

Notes:

  • The response contains a single movie object with information such as movie ID, title, genre, and release year.
  • If the movie with the specified title is not found, a 404 Not Found error is returned with an error message.

User Registration

Description: Registers a new user.

Endpoint URL: /users/register

Method Type: POST

Query Parameters: None

Request Body:

Format: JSON
  Example:
  {
    "username": "example_user",
    "email": "user@example.com",
    "password": "secure_password"
  }

Response Data:

Format: JSON
  Example:
  {
    "message": "User accepted."
  }

Status Codes:

  • 201 Created: Successful user registration, returns a confirmation message.

Notes:

  • The request should include a JSON body with the user's username, email, and password.
  • The response contains a confirmation message indicating the successful user registration.
